# FBX Phase 0 Corpus

This folder is the committed phase-0 corpus contract for the native FBX importer/exporter effort.

- `fbx-corpus.manifest.json` is the source of truth for checked-in, synthetic malformed, and planned coverage entries.
- Checked-in importable assets must also have a committed `*.summary.json` golden file generated by the baseline harness.
- Planned entries represent required coverage that is not yet checked into the repo because of size, licensing, or sourcing constraints.
- Synthetic malformed fixtures exist so parser validation can fail closed on obviously bad input without depending on third-party samples.

Generate or refresh the checked-in golden summaries with:

```powershell
dotnet run --project XREngine.Benchmarks -- --fbx-phase0-report
```

Current checked-in corpus reality:

- The repo currently contains large ASCII FBX Sponza fixtures that are useful as semantic baselines.
- The binary 7.4/7.5 tokenizer corpus required for the native parser is tracked as planned entries in the manifest and still needs to be sourced.
