# XRENGINE Developer Guides

[Back to docs index](../README.md)

Developer guides explain implemented engine features from the code side. Use this section when you need to integrate with a subsystem, call its APIs, extend behavior, inspect diagnostics, or find the source and tests that enforce a feature contract.

These docs are intentionally closer to the code than the user guide. They should describe runtime behavior, important types, settings objects, extension points, test coverage, diagnostics, and implementation references.
